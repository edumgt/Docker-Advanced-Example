# 평가/산출물 가이드

## 평가 비율(예시)
- 일일 실습 과제: 40%
- 중간 점검(이미지/Compose 품질): 20%
- 캡스톤 프로젝트: 40%

## 제출물(권장)
- 실습 로그(명령어 기록), 스크린샷/캡처(선택)
- 각 Day 과제 README 또는 md 문서
- 캡스톤: `README.md`, `docker-compose.yml`, `Dockerfile`, `Jenkinsfile`, `runbook.md`

## 체크리스트(핵심)
- [ ] 클린 환경에서 재현 가능한가?
- [ ] 컨테이너 종료/재시작/로그 확인이 자연스러운가?
- [ ] 데이터 영속성(볼륨) 설계가 되어 있는가?
- [ ] 포트 노출 최소화 및 네트워크 분리가 되어 있는가?
- [ ] 이미지 크기/레이어 최적화가 반영되어 있는가?
- [ ] CI에서 빌드/테스트/푸시가 자동화되어 있는가?
- [ ] 장애 2개 이상 재현/대응 Runbook이 있는가?
