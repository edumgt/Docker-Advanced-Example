# 강사용 진행 가이드(Instructor Guide)

## 운영 원칙
- “설명 20분 → 실습 40~60분 → 리뷰 10분” 리듬을 유지합니다.
- 장애 대응/디버깅은 반드시 **재현 가능한 시나리오**로 체득하게 합니다.
- 실습 시, 결과물은 “스크린샷”보다 **명령어 로그 + README**를 우선합니다.

## 시간 운영 예시(7시간 기준)
- 1교시(90m): 개념 + 데모
- 2교시(90m): 실습(기본)
- 3교시(90m): 실습(심화/확장)
- 4교시(60m): 과제/리뷰/정리 + Q&A

## 흔한 문제와 대응
- Windows/WSL 경로/권한 이슈 → WSL 내부 폴더에서 실습 유도
- 포트 충돌(80/443/8080) → `lsof/ss/netstat`로 점검 습관
- 이미지 아키텍처 불일치(arm64/amd64) → `--platform` 설명
